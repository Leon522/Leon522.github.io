跨域请求的几种方式

发表于 2016-05-21   |     |   阅读次数

跨域：Cross Domain Request

请求不同的域上的资源(HTML/CSS/JS/SCRIPT/JPEG等)
不同的域可能是：

  1. 不同的协议
  2. 不同的域名
  3. 不同的端口号
  4. 默认情况下，浏览器可以针对HTML、CSS、JS、图片发起跨域请求——同步请求。
  5. 客户端浏览器默认禁用XHR的跨域请求（不论何种资源）——出于安全考虑！
    使用场景：项目很大，于是把静态HTML放在A服务器，动态数据(PHP文件)放在B服务器，动态数据(JSP文件)放在C服务器；A服务器上的静态HTML需要动态数据，必需使用异步请求——不能使用XHR发起跨域请求。

**解决办法一： JSONP

JSON with Padding，JSONP，填充式JSON，是以一种手段、一种技巧，可以巧妙的跨过XHR跨域请求的限制——与JSON不同，不是一种字符串格式。
思路：

  1. 不使用XHR对象——XHR的跨域请求会被浏览器禁止！

  2. 在DOM树动态添加script元素——SCRIPT的跨域请求浏览器允许

    3.PHP页面返回的JSON字符串稍加改造：

    $json = ‘{“ename”:”Tom”, “age”:20}’;
    //echo $json;
    echo ‘doResponse(‘. $json .’)’; //字符串

4.在静态HTML页面中声明function doResponse(data){ …}即可
客户端最终执行的语句如下：

doResponse({"ename":"Tom", "age":20})

其中的函数本地在HTML中声明，其中的数据是PHP返回的。

由于服务器不知道客户端的回调是什么，不可能hard code成jsonpcallback，所以就带一个QueryString让客户端告诉服务端，回调方法是什么，当然，QueryString的key要遵从服务端的约定，上面的是”callback“。返回的就是对一个方法的回调，参数就是需要请求的数据


AJAX和JSONP的关系？
严格说，二者没有任何关系！
AJAX的核心是XHR对象，不能发起跨域请求；
JSONP的核心是SCRIPT元素，可以发起跨域请求；
二者的相似点：都能实现异步请求，异步获取服务器端数据。

2.jQuery对JSONP的支持
jQuery对AJAX的封装函数：

  1. $(…).load( )
  2. $.get( )
  3. $.post( )
  4. $.getScript( )
  5. $.getJSON( ) 使用XHR对象发起异步请求
  6. $.ajax( ) 使用XHR对象发起异步请求

jQuery对JSONP的封装函数：

  1. $.getJSON( ) 使用动态SCRIPT元素发起异步请求

    $.getJSON('跨域地址/xx.php?callback=?', function(data){  })

  2. $.ajax( ) 使用动态SCRIPT元素发起异步请求

    $.ajax({
url: '跨域地址/xx.php',
dataType: 'jsonp',
success: function(data){  }
});

缺点：
1、这种方式无法发送post请求，无法解决数据量大的情况
2、另外要确定jsonp的请求是否失败并不容易，大多数框架的实现都是结合超时时间来判定。
3、安全隐患（容易受到csrf攻击，csrf的解决必须是post请求配合token使用）

**解决方法二：CORS（跨域资源共享，Cross-Origin Resource Sharing）

CORS的原理：
W3C制定的跨站资源分享标准。post前会产生一次options嗅探来确认有否跨域请求的权限；客户端post时会带上Origin头指示来源网站，服务端响应时需带上Access-Control-Allow-Origin头与Origin头的值匹配，以示许可。ie8提供了封装好的XDomainRequest对象，部分实现了该标准；而其它浏览器则提供了XMLHttpRequest（Level 2）对象。
CORS定义一种跨域访问的机制，可以让AJAX实现跨域访问。CORS 允许一个域上的网络应用向另一个域提交跨域 AJAX 请求。实现此功能非常简单，只需由服务器发送一个响应标头即可。


“*”号表示允许任何域向我们的服务端提交请求

也可以指定域名
这种方式弥补了JSONP在数据量大，get方式搞不定的缺陷

**解决方法三 Proxy代理

这种方式首先将请求发送给后台服务器，通过服务器来发送请求，然后将请求的结果传递给前端。
我们不能在浏览器端直接使用AJAX来跨域访问资源，但是在服务器端是没有这种跨域安全限制的。
北京服务器(域名:www.beijing.com)
上海服务器(域名：www.shanghai.com)
比如在北京的web服务器的后台(www.beijing.com/proxy.php)来调用上海服务器(www.shanghai.com/services.php)的服务，然后再把访问结果返回给前端，这样前端调用北京同域名的服务就和调用上海的服务效果相同了。

JavaScript中的参数传递

发表于 2016-04-10   |     |   阅读次数

JavaScript中的数据结构

1.原始数据类型值 （primitive type）
如undefind，null，boolean，number，string。
2.引用数据类型值（Object type）
如Object，array，function，date等。

声明变量时不同的内存分配
1.原始值：存储在栈内存(stack)中，值直接存储在变量访问的地方。原始数据类型占据的空间是固定的，所以可将他们存储在较小的内存空间，便于迅速查找变量的值。
2.引用值：存储在堆内存(heap)中，存储在变量处的值是一个指针，指向存储对象的内存地址。因为应用类型值大小会发生改变，所以不能够放在栈中。所以访问的时候首先得到的是它在堆内存中的地址，再按地址去得到对象的值。

复制变量时的不同
1.原始值：将一个保存着原始值的变量复制给另一个变量时，会将原始值得副本复制给新变量，此后两个变量是完全独立的，只有value是相同的，
2.引用值：将一个保存着对象内存地址的引用变量复制给另一个变量时，会把地址赋值给新变量，也就是说这两个变量同时指向了堆内存的同一个对象，他们中任何一个的改变都会反应在另一个身上。
//**

var obj1 = {value:"111"};

var obj2 = {value:"222"};

function changeStuff(obj){
    obj.value="333"
    obj = obj2;
    console.log(obj.value);
}
changeStuff(obj1);//"222" 参数obj指向了新的对象obj2
console.log(obj1.value);//"333" obj1任然指向原来的对象，只是value改变了

//**

var arr = [1,2,3,4,5];
function fun(obj){
    obj[0] = 6;
    obj = "hi";
    console.log(obj);  
}
fun(arr);//"hi"  原来obj指向arr对象的地址变量被赋值为变量"hi"
console.log(arr);//Array [ 6, 2, 3, 4, 5 ]  arr任然指向原来的对象，只是对象的值被另一个引用改变了

//**

var arr = [1,2,3,4,5];
function fun(){
    arr[0] = 6;
    arr = "hi";
    console.log(arr);  
}
fun();//hi  函数内调用全局arr，arr存有对象的地址已经被赋值为"hi"
console.log(arr);//"hi" 函数内改变的是全局变量,内存已经没有存放对象地址的变量了，所以对象会被回收。

//**

var arr = [1,2,3,4,5];
function fun(obj){
    arr[0] = 6;
    arr = "hi";
    console.log(obj);  
}
fun(arr);//Array [ 6, 2, 3, 4, 5 ]   对象的值已经被被另一个引用改变了
console.log(arr);//"hi" arr存放指向对象地址的变量被赋值为变量"hi"

当改变对象内部的属性的时候，这个改变会体现在外部，因为他们共同的对象被改变了。
而如果在函数内部给这个参数赋值另一个对象时，这个参数就会更改它的值为新对象的内存地址指向新对象，但是原来的变量任然指向原来的对象，此时是相互独立的。
