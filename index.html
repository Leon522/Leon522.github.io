JavaScript中的参数传递

发表于 2016-04-10   |     |   阅读次数

JavaScript中的数据结构

1.原始数据类型值 （primitive type）
如undefind，null，boolean，number，string。
2.引用数据类型值（Object type）
如Object，array，function，date等。

声明变量时不同的内存分配
1.原始值：存储在栈内存(stack)中，值直接存储在变量访问的地方。原始数据类型占据的空间是固定的，所以可将他们存储在较小的内存空间，便于迅速查找变量的值。
2.引用值：存储在堆内存(heap)中，存储在变量处的值是一个指针，指向存储对象的内存地址。因为应用类型值大小会发生改变，所以不能够放在栈中。所以访问的时候首先得到的是它在堆内存中的地址，再按地址去得到对象的值。

复制变量时的不同
1.原始值：将一个保存着原始值的变量复制给另一个变量时，会将原始值得副本复制给新变量，此后两个变量是完全独立的，只有value是相同的，
2.引用值：将一个保存着对象内存地址的引用变量复制给另一个变量时，会把地址赋值给新变量，也就是说这两个变量同时指向了堆内存的同一个对象，他们中任何一个的改变都会反应在另一个身上。
//**

var obj1 = {value:"111"};

var obj2 = {value:"222"};

function changeStuff(obj){
    obj.value="333"
    obj = obj2;
    console.log(obj.value);
}
changeStuff(obj1);//"222" 参数obj指向了新的对象obj2
console.log(obj1.value);//"333" obj1任然指向原来的对象，只是value改变了

//**

var arr = [1,2,3,4,5];
function fun(obj){
    obj[0] = 6;
    obj = "hi";
    console.log(obj);  
}
fun(arr);//"hi"  原来obj指向arr对象的地址变量被赋值为变量"hi"
console.log(arr);//Array [ 6, 2, 3, 4, 5 ]  arr任然指向原来的对象，只是对象的值被另一个引用改变了

//**

var arr = [1,2,3,4,5];
function fun(){
    arr[0] = 6;
    arr = "hi";
    console.log(arr);  
}
fun();//hi  函数内调用全局arr，arr存有对象的地址已经被赋值为"hi"
console.log(arr);//"hi" 函数内改变的是全局变量,内存已经没有存放对象地址的变量了，所以对象会被回收。

//**

var arr = [1,2,3,4,5];
function fun(obj){
    arr[0] = 6;
    arr = "hi";
    console.log(obj);  
}
fun(arr);//Array [ 6, 2, 3, 4, 5 ]   对象的值已经被被另一个引用改变了
console.log(arr);//"hi" arr存放指向对象地址的变量被赋值为变量"hi"

当改变对象内部的属性的时候，这个改变会体现在外部，因为他们共同的对象被改变了。
而如果在函数内部给这个参数赋值另一个对象时，这个参数就会更改它的值为新对象的内存地址指向新对象，但是原来的变量任然指向原来的对象，此时是相互独立的。
